# FnMediaCoverGenerator 风格 5（幕光拼贴）设计规格

## 1. 文档目标

本文档定义 `plugins.v2/fnmediacovergenerator` 的新静态封面风格 `static_5`。  
该风格的视觉方向已经在头脑风暴阶段确定为：

- 总体风格：`A / Cinematic Veil`
- 细化方向：`A2 / Hybrid Collage`
- 气质定位：介于「克制的流媒体质感」与「浓烈的影院海报感」之间

本文档用于后续实现计划和开发落地，不包含代码实现。

## 2. 背景与问题

当前插件的静态风格主要覆盖：

- `static_1`：多图拼贴
- `static_2`：单图风格
- `static_3`：九宫格拼贴
- `static_4`：单图简化风格

围绕飞牛媒体库封面生成，现阶段存在以下痛点：

1. 风格 3 的九宫格在素材不足或补图逻辑不理想时，容易出现重复感。
2. 纯拼贴风格能体现媒体库丰富度，但缺少专业流媒体封面的主视觉重心。
3. 纯单图风格虽然稳，但会削弱「这是一个媒体库集合」的表达。
4. 长标题、亮底图和复杂背景下的文字可读性仍依赖人工调参，不够稳。

本次新增风格的目标，是在不改动现有风格视觉规则的前提下，提供一套更接近专业流媒体主视觉、同时保留媒体库集合感的静态封面生成方案。

## 3. 设计结论

### 3.1 风格名称

- 内部编号：`static_5`
- 对外展示：`风格 5（幕光拼贴）`

### 3.2 核心设计原则

1. 画面重心必须明确，不能退化成信息均分的拼贴墙。
2. 素材池优先使用飞牛媒体库条目图，避免把媒体库默认 `image_list` 和条目补图混用后再次引入重复。
3. 标题阅读优先级高于所有特效，任何视觉增强都不能以牺牲可读性为代价。
4. 首版优先把默认成片质量做高，不引入大量新参数和高级开关。

### 3.3 最终选定方案

新风格采用 `1 张主图 + 0~3 张辅图` 的轻拼贴结构：

- 主图承担电影感和视觉重心。
- 辅图承担媒体库集合感和随机性表达。
- 标题落在底部安全区，并通过智能遮罩与智能锚点保持可读性。

## 4. 范围与非目标

### 4.1 本次范围

1. 新增一个新的静态风格 `static_5`。
2. 在插件配置页和预览卡中增加风格 5 的入口。
3. 为风格 5 新增专属的素材筛选、抽图、排版和渲染规则。
4. 为风格 5 增加「尽量避开上次主图」的轻量状态记忆。
5. 为风格 5 增加更明确的日志输出，便于判断素材数量、退化模式和随机效果。

### 4.2 明确不在本次范围内

1. 不修改 `static_1` 到 `static_4` 的视觉规则。
2. 不修改动画风格的视觉规则。
3. 不引入人脸识别避让。
4. 不引入拖拽式版式编辑。
5. 不增加新的高级调参面板。
6. 不在首版实现「少图时复用辅图并做人眼难辨差异化」策略。

说明：头脑风暴早期曾讨论过在素材不足时复用辅图，但最终确认的首版策略是「少图时减少辅图数量，不硬凑满」，因此该策略明确排除在本次实现范围外，以避免文档与实现口径冲突。

## 5. 用户可见效果

### 5.1 画面骨架

风格 5 的标准形态如下：

- 主图：1 张，整张铺设，为第一视觉。
- 辅图：3 张，位于主图下方或下侧的低权重缩略带，不抢主图。
- 文字区：固定在底部安全区，默认左对齐。
- 遮罩层：从底部向上的黑色渐变遮罩，用于保证文字可读性。

### 5.2 气质控制

整体气质按以下比例控制：

- 流媒体克制感：60%
- 影院海报戏剧感：40%

具体体现为：

- 不使用夸张的金属字和厚重描边。
- 不使用高饱和度的花哨滤镜。
- 主图保留大画面氛围，但辅图仍明确提示「这是一个媒体库集合」。

### 5.3 文字层结构

文字区采用三层结构：

1. 小眉标（Eyebrow）
2. 主标题（中文主标题）
3. 可选副标题（英文副标题）

数据来源规则如下：

- 主标题：沿用当前配置中的中文标题，即 `__get_title_from_config()` 返回的 `zh_title`。
- 副标题：沿用当前配置中的英文标题或默认英文映射，即 `__get_title_from_config()` 返回的 `en_title`。
- 小眉标：根据媒体库类型或名称语义生成固定英文短语。

小眉标的首版规则：

- 可识别为电影库时，使用 `Movie Library`
- 可识别为剧集库时，使用 `Series Collection`
- 可识别为动画相关媒体库时，使用 `Animation Archive`
- 其他情况使用 `Featured Collection`

## 6. 代码触点与职责划分

### 6.1 `plugins.v2/fnmediacovergenerator/__init__.py`

职责：

1. 注册 `static_5` 对应的风格入口。
2. 更新配置项和预览卡，使 UI 能显示风格 5。
3. 为风格 5 定义素材需求和素材来源策略。
4. 管理风格 5 的「上次主图 URL」状态。
5. 记录风格 5 的专属日志。

预期改动点：

- 增加 `style_static_5` 的导入。
- 更新风格选择项，从 4 个静态风格扩展到 5 个。
- 更新 `__get_cover_style_parts()` 和 `__style_preview_src()` 的最大值上限。
- 为 `__get_required_items()` 增加 `static_5` 规则。
- 在 `_generate_image_from_path()` 中增加 `static_5` 分支。
- 增加风格 5 所需的状态读写辅助方法。

### 6.2 `plugins.v2/fnmediacovergenerator/style/style_static_5.py`

职责：

1. 读取本地已准备好的封面素材。
2. 根据素材数量确定标准模式或退化模式。
3. 完成主图布局、辅图布局、遮罩、文字、字效与导出。

建议内部职责拆分：

- `load_cover_sources()`：读取可用图片并做顺序管理。
- `choose_text_anchor()`：在底部左侧或右侧安全区中选择更适合的文字锚点。
- `build_dynamic_mask()`：生成遮罩层。
- `enhance_primary_image()`：对主图做轻量增强。
- `render_primary_and_echoes()`：渲染主图与辅图。
- `draw_title_block()`：绘制小眉标、主标题、副标题。
- `render_style_static_5()`：对外暴露统一入口。

### 6.3 预览素材

需要补充风格 5 的预览图资源，并保持与现有静态风格预览加载方式一致。  
如果现有预览图逻辑依赖固定索引范围，则需要同步扩展到 5。

## 7. 数据流设计

### 7.1 总流程

风格 5 的生成流程如下：

1. 读取媒体库基本信息。
2. 获取媒体库条目图候选列表。
3. 对候选 URL 做标准化和去重。
4. 结合「上次主图 URL」状态做随机无放回抽样。
5. 下载抽中的图片，并继续做内容级去重。
6. 将图片落盘到临时目录。
7. 根据最终落盘数量进入标准模式或退化模式。
8. 渲染封面。
9. 保存本地封面并按原逻辑决定是否上传。
10. 在成功生成后更新「上次主图 URL」状态。

### 7.2 素材来源规则

风格 5 的素材来源采用「条目图优先，媒体库图仅在条目图为空时兜底」策略：

1. 首选飞牛条目补图结果。
2. 若条目补图去重后数量大于 0，则仅使用条目补图，不再混入媒体库默认 `image_list`。
3. 若条目补图去重后数量为 0，且媒体库默认 `image_list` 有可用内容，则退回使用 `image_list`。
4. 若两者都为空，则跳过该媒体库并记录明确日志。

该策略用于彻底避免「4 张媒体库图 + N 张条目图」混合后再度出现重复感。

### 7.3 URL 去重与下载后去重

去重分两层进行：

1. URL 层去重  
   在进入抽样前，对 URL 做标准化并去重，避免相同链接重复参与随机。

2. 内容层去重  
   下载后继续基于内容哈希去重，避免不同 URL 指向同一张图。

最终用于渲染的数量，以「下载并通过内容去重后的有效图片数」为准。

### 7.4 抽图规则

风格 5 的抽图规则如下：

1. 从去重后的候选池中随机无放回抽取最多 4 张。
2. 第 1 张为主图。
3. 第 2 至第 4 张为辅图。
4. 每次生成都重新随机。

为增强连续多次生成时的变化感，新增「尽量避开上次主图」规则：

1. 按媒体库维度记录上一次成功生成时的主图 URL。
2. 本次抽主图时，优先从不等于上次主图 URL 的候选中选择。
3. 如果候选池在排除后为空，则放开限制，允许再次使用。
4. 该规则只影响主图，不强制影响辅图。

### 7.5 状态存储规则

新增插件持久化数据键：

- `style5_last_primary_urls`

建议结构：

```json
{
  "飞牛影视::094d1614638642c78d48318fa5625a8c": "https://example.com/cover-a.jpg",
  "飞牛影视::7da518a22d9a4e688069185c08905c7c": "https://example.com/cover-b.jpg"
}
```

键名使用：

- `"{server_name}::{library_id}"`

更新时机：

- 仅在封面成功生成并完成本地保存后更新。

容量控制：

- 首版建议保留最近 200 个媒体库键值，超出后按插入顺序裁剪。

### 7.6 退化模式

风格 5 的退化路径固定如下：

- 有效图片数 `>= 4`：标准 A2 模式，`1 主图 + 3 辅图`
- 有效图片数 `= 3`：`1 主图 + 2 辅图`
- 有效图片数 `= 2`：`1 主图 + 1 辅图`
- 有效图片数 `= 1`：自动退化为 A1 单主图幕光模式
- 有效图片数 `= 0`：终止生成并记日志；若条目图为空但 `image_list` 可用，则先执行来源兜底再判断

首版不允许为了凑满版式而复制辅图。

## 8. 版式与视觉规则

### 8.1 主图与辅图的关系

主图规则：

- 占据绝大部分视觉面积。
- 允许使用整图裁切，但不得进行明显变形。
- 优先保留人物、主体或氛围感区域。

辅图规则：

- 作为低权重「回声带」出现。
- 使用较小尺寸和统一圆角。
- 允许整体轻微压暗，防止抢主图。
- 允许统一冷暖倾向，但不做重滤镜。

### 8.2 文字锚点

文字区固定落在底部安全区，但左右位置允许智能选择。

首版算法：

1. 在主图底部取左下和右下两个候选安全区。
2. 分别计算亮度均值和局部复杂度。
3. 优先选择亮度更低、复杂度更低的一侧作为文字锚点。
4. 若两侧都不理想，则保持左对齐并增强底部遮罩。

首版不做人脸识别避让。

### 8.3 遮罩规则

遮罩为从下到上的黑色渐变遮罩，用于托住文字。

基本规则：

- 方向：`bottom -> top`
- 透明度范围：约 35% 到 60%
- 调整依据：底部安全区亮度越高，遮罩越强

当主图本身偏亮时，还允许在主图区额外加一层轻压暗蒙版，但不能让整张图显脏。

### 8.4 标题排版

排版规则：

1. 主标题固定为第一视觉。
2. 默认左对齐，不做居中排版。
3. 短标题整体上移少量，增加大气感。
4. 长标题通过测量文本宽度后动态缩小字号，不缩放画布。
5. 主标题最多允许两行。

首版字号策略：

- 从配置字号开始测量。
- 若超出安全区宽度，则逐级减小字号。
- 缩放下限约为基础字号的 88%。

### 8.5 副标题与小眉标

副标题规则：

- 使用英文副标题。
- 作为低于主标题一级的信息层。
- 若为空，则不占位。

基线处理规则：

- 不开放新滑块。
- 首版通过内置偏移值校准中英文视觉基线。

小眉标规则：

- 使用较小字号和较大字间距。
- 主要承担氛围标签作用，而非信息主体。

### 8.6 字效规则

主标题字效采用微浮雕方向：

- 主体颜色：白色或接近白色
- 顶部高光：极轻
- 投影：深色软阴影
- 描边：默认不启用

目标效果是「像专业流媒体封面标题」，而不是「游戏海报特效字」。

### 8.7 轻量增强

主图增强首版只允许以下轻量处理：

- 对比度：约 `+10%`
- 饱和度：约 `+8% ~ +12%`
- 锐化：极轻微

辅图增强必须弱于主图。

## 9. 实现策略约束

### 9.1 与当前流程的兼容方式

风格 5 应尽量复用当前插件已有能力：

1. 复用当前飞牛媒体库枚举流程。
2. 复用当前下载、临时目录落盘和内容哈希去重逻辑。
3. 复用当前标题配置和英文副标题映射逻辑。
4. 复用当前封面保存、历史记录和自动上传流程。

### 9.2 需要改变的策略

仅对风格 5 调整以下策略：

1. 素材来源优先级从「媒体库图 + 条目补图混合」切换为「条目图优先、媒体库图仅空池兜底」。
2. 图片需求从现有风格 3 的 `9` 张降为最多 `4` 张。
3. 引入按媒体库维度保存的「上次主图 URL」状态。

## 10. 日志与可观测性

风格 5 需要明确记录以下信息：

1. 条目图原始返回数量
2. URL 去重后数量
3. 下载成功数量
4. 内容去重后有效数量
5. 最终抽中数量
6. 是否命中「避开上次主图」
7. 最终使用的模式：标准 A2 / 减辅图模式 / A1 退化模式 / 跳过
8. 若触发来源兜底，需要明确记录来源切换为 `image_list`

日志目标是让用户一眼看出：

- 飞牛到底返回了多少张
- 插件到底用了多少张
- 为什么这次看起来和上次不同或相似

## 11. 错误处理

### 11.1 可恢复错误

以下情况应记录警告并继续：

1. 个别图片下载失败
2. 个别 URL 下载后内容重复
3. 条目图数量不足，触发减辅图模式

### 11.2 不可恢复错误

以下情况应终止该媒体库生成并返回失败结果：

1. 条目图和兜底 `image_list` 都为空
2. 下载后有效图片数为 0
3. 渲染阶段抛出异常
4. 字体不可用

### 11.3 用户可见提示

对用户可见的失败信息应尽量直接：

- `媒体库没有可用的条目封面源图`
- `媒体库封面素材不足，已退化为单主图模式`
- `媒体库封面生成失败`

首版不追求非常细碎的错误分类，但要保证文案能解释结果。

## 12. 测试策略

### 12.1 单媒体库视觉验证

至少验证以下场景：

1. 有 20+ 张条目图的电影库
2. 有 10+ 张条目图的剧集库
3. 仅有 3 张有效条目图的媒体库
4. 仅有 1 张有效条目图的媒体库
5. 条目图为空但 `image_list` 有值的媒体库

### 12.2 连续生成验证

对同一个媒体库连续生成 2 到 3 次，验证：

1. 主图大概率发生变化
2. 辅图组合不固定
3. 日志中能看出是否命中「避开上次主图」

### 12.3 文字可读性验证

至少验证：

1. 短中文标题
2. 长中文标题
3. 有英文副标题
4. 无英文副标题
5. 底部偏亮主图

### 12.4 回归验证

必须确认：

1. `static_1` 到 `static_4` 不受影响
2. 动画风格不受影响
3. 自动上传流程不受影响
4. 历史封面保存和展示不受影响

## 13. 验收标准

满足以下条件即可认为风格 5 首版达标：

1. 能在配置页稳定选择和预览 `风格 5（幕光拼贴）`
2. 同一媒体库连续生成 2 到 3 次时，主图大概率不同
3. 长标题不溢出，短标题不显空
4. 亮底图上的白字仍保持较高可读性
5. 画面观感更接近「媒体库电影感封面」，而不是「几张图简单拼接」
6. 现有风格与上传流程不回归

## 14. 实施顺序建议

后续实现计划建议按以下顺序拆分：

1. 注册风格 5 与配置入口
2. 调整风格 5 的素材来源与抽图逻辑
3. 实现 `style_static_5.py` 基础版式
4. 接入动态遮罩与智能锚点
5. 接入「避开上次主图」状态
6. 补日志与回归验证

---

本规格的唯一目标是定义首版 `static_5`。  
任何超出本文档范围的增强项，都应在后续独立迭代中追加，而不是在首版实现中顺手混入。
